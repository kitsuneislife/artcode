# Registro de Decisões de Design

Este diretório contém decisões formais do projeto (alternative: ADR style) e o histórico de escolhas arquiteturais.

Formato sugerido para cada decisão:
- Título: curto e descritivo
- Data: YYYY-MM-DD
- Proponents: nomes
- Contexto: problema/constraints
- Decision: resumo da escolha
- Consequências: trade-offs e ações futuras

Use o template canônico em `docs/decisions/0000-template.md`.

Ao tomar uma decisão que altera arquitetura ou contrato público:
1. Crie um novo arquivo `docs/decisions/NNNN-...md` a partir do template.
2. Preencha contexto, decisão e consequências com trade-offs explícitos.
3. Linke a RFC e/ou PR relacionada para manter rastreabilidade.
